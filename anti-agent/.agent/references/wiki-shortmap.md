### [schema.md]
# LLM Wiki Schema ## Purpose This schema defines how the LLM Wiki is structured, maintained, and consumed by Antigravity Agents. ## Structure 1. **Raw Sources**: Documentation in `.agent/references/` or project `docs/`. These are immutable sources of truth. 2. **Wiki Layer (`/wiki/`)**: Markdown files generated and maintained by the LLM. 3. **Shortmap Layer (`.agent/references/wiki-shortmap.md`)**: A token-efficient version of the wiki for agent context. ## Conventions - **No Icons**: Avoid using emojis or icons in the Shortmap layer to save tokens. - **Structure**: - `## Summary`: Persistent overview of the topic. - `## Links`: Cross-references to other wiki pages. - `## Ref`: Links back to raw sources. - **Logic**: Incremental synthesis. When new info arrives, update relevant pages, do not just append. ## Workflows - **Ingest**: Process a new source ?Update `index.md` ?Update/Create entity pages ?Log in `log.md`. - **Minify**: Run `.agent/scripts/skill-reducer.ps1` to rebuild the `wiki-shortmap.md`. - **Prune**: Periodically check for stale info or dead links. ## Metadata Updated: 2026-04-17 Maintainer: Orchestrator

### [logic\code_patterns.md]
# Code Patterns & Architectural Standards This document defines the prescribed coding patterns for the project to ensure total consistency between "Parent Source" logic and Agent-implemented features. ## 1. Composable-First Logic - **Rule**: All business logic, state management, and API orchestration must reside in **Composables** (`src/composables/`). - **Rationale**: Keeps `.vue` files focused on Template and UI state. Facilitates cross-component logic sharing. - **Pattern**: - Define a `state` object using `reactive()` (outside the function for shared/singleton state, inside for local state). - Provide an `initialState...` factory function. - Export a `useModuleName` function returning `state` and methods. ## 2. Configuration Management - **Rule**: Avoid inline configurations (arrays, maps) for complex UI components like Tables or Charts. - **Location**: Use `src/pages/[module]/configs/`. - **Example**: `HospitalColumns.js`, `DashboardCharts.js`. ## 3. UI Polish & Animations - **Rule**: Use subtle micro-animations (ex:, `fadeIn`) to improve UX feel, but ensure they don't block interaction. - **Style**: Scoped SCSS in `.vue` files is preferred. ## 4. Agentic Footprint Minimization - **Strictly Prohibited**: Decorative emojis/icons in code comments or documentation. - **Required**: Maintain a minimalist, professional tone. If a pattern is detected as "Agent-only" (ex:, inline logic for convenience), it must be refactored to match the parent source.
